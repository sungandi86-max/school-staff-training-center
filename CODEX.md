# Codex 작업 원칙

이 저장소는 학교별 실사용 데이터를 다루는 교직원 교육 운영센터 템플릿입니다. 작업할 때는 개인정보 보호와 학교별 복사 사용 가능성을 가장 먼저 고려합니다.

## 작업 전 문서 확인

- 작업 전 `README.md`, `NEXT.md`, `DESIGN.md`, 관련 `docs/` 문서를 먼저 확인합니다.
- 시트 구조를 바꾸는 작업은 `docs/sheet-structure.md`를 먼저 확인합니다.
- 개인정보 흐름을 바꾸는 작업은 `docs/privacy-policy.md`와 `docs/architecture.md`를 먼저 확인합니다.

## 기존 기능 보존

- 기존 시트 탭, 필드명, Apps Script 액션, Vercel 화면 계약을 함부로 변경하지 않습니다.
- 변경이 필요하면 관련 문서를 함께 수정합니다.
- 학교별 템플릿 파일은 삭제하거나 이동하지 않습니다.

## 민감정보 서버 저장 금지

- Vercel 서버에 교직원 명단, 인증코드, 출석기록, 전자서명 이미지를 저장하지 않습니다.
- 샘플 데이터에는 실제 교직원 개인정보를 넣지 않습니다.
- 로그와 에러 메시지에 민감정보가 남지 않도록 합니다.

## Apps Script와 Vercel 역할 분리

- Apps Script는 Google Sheet 읽기/쓰기, 대상자 확인, 중복 출석 확인, Drive 저장, 최종 서명부 생성을 담당합니다.
- Vercel은 화면 제공과 Apps Script API 호출만 담당합니다.
- 민감정보의 원본 저장 위치는 학교별 Google Sheet와 Google Drive입니다.

## 새 패키지 설치 금지 원칙

- 초기 문서/구조 정리 단계에서는 새 패키지를 설치하지 않습니다.
- 기능 구현 단계에서도 기존 의존성으로 해결 가능한지 먼저 확인합니다.
- 새 패키지가 꼭 필요하면 이유와 대안을 문서화한 뒤 진행합니다.

## 검증 명령

웹앱이 생성된 뒤에는 가능한 경우 다음 순서로 확인합니다.

```bash
npm run lint
npm run typecheck
npm run build
```

문서만 수정한 경우에는 빌드 명령을 실행하지 않아도 되며, 실행하지 않은 이유를 작업 보고에 남깁니다.
