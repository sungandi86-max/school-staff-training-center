# Google Sheet 탭 구조

학교 교직원 교육센터는 학교별 Google Sheet를 운영 데이터 저장소로 사용합니다. 교직원 명단, 교육대상, 출석기록, 전자서명 기록은 GitHub Pages나 Vercel에 저장하지 않고 이 Sheet와 학교별 Google Drive에만 저장합니다.

## 기본 탭 구조

```text
00_학교설정
01_교육목록
02_교직원명단
03_교육대상
04_QR출석기록
05_전자서명기록
06_이수증제출
07_내이수현황
08_최종서명부
99_코드값
```

## 연결 원칙

- 모든 교육 데이터는 `교육ID`를 기준으로 연결합니다.
- 모든 교직원 데이터는 `교직원ID`를 기준으로 연결합니다.
- `01_교육목록`과 `02_교직원명단`은 직접 연결하지 않습니다.
- 특정 교육의 대상자는 반드시 `03_교육대상`에서 판단합니다.
- 전자서명 이미지는 Google Drive 지정 폴더에 저장하고 Sheet에는 파일 ID와 URL만 기록합니다.
- Apps Script API 응답은 화면에 필요한 최소 정보만 반환합니다.
- 전체 교직원 명단을 GitHub Pages 화면으로 내려보내지 않습니다.

## 00_학교설정

학교별 설정값을 관리합니다.

| 필드 | 설명 |
| --- | --- |
| 설정키 | 설정 항목 이름 |
| 설정값 | 학교별 설정값 |
| 설명 | 관리자용 설명 |

필수 설정값:

```text
학교명
교육센터명
학교로고URL
메인컬러
보조컬러
담당부서명
담당자명
담당자 연락처
전자서명 저장 폴더 ID
최종 서명부 저장 폴더 ID
개인정보 안내문
```

```text
장소

```

## 02_교직원명단

교직원 인증과 기본 소속 정보를 저장합니다.

```text
교직원ID
성명
부서
직위
인증코드
재직상태
비고
```

주의:

- 전체 명단을 GitHub Pages로 내려보내지 않습니다.
- 인증 성공 시에도 화면에 필요한 최소 정보만 반환합니다.

## 03_교육대상

교육별 대상자를 판단하는 연결 탭입니다.

```text
교육ID
교직원ID
대상여부
서명제외여부
필수여부
비고
```

판단 기준:

- `대상여부`가 참인 경우에만 출석 대상자로 처리합니다.
- `서명제외여부`가 참이면 교육 자체가 전자서명을 요구해도 해당 교직원은 서명 없이 출석할 수 있습니다.
- `필수여부`는 내 이수현황에서 필수 교육 표시 기준으로 사용합니다.

## 06_이수증제출

외부 연수 또는 온라인 연수 이수증 제출 기능을 위한 탭입니다. 1차 MVP에서는 새로 확장하지 않고, 기존 기능이 있을 경우 유지 대상으로 둡니다.

```text
제출ID
교육ID
교직원ID
성명
파일ID
파일URL
제출일시
처리상태
검토자
검토일시
비고
```

## 07_내이수현황

교직원이 본인 인증 후 확인하는 이수현황 기준 탭입니다. 실제 구현에서는 교육목록, 교육대상, 출석기록, 전자서명기록을 조합해 계산하거나 View 성격으로 관리할 수 있습니다.

```text
교육ID
교직원ID
필수여부
이수완료여부
서명완료여부
출석일시
비고
```

## 구조 점검 체크리스트

- 필수 탭이 모두 존재하는지 확인합니다.
- 각 탭의 첫 행 필드명이 문서와 일치하는지 확인합니다.
- `교육ID`와 `교직원ID`가 비어 있지 않은지 확인합니다.
- `03_교육대상`에 교육별 대상자가 등록되어 있는지 확인합니다.
- 전자서명 저장 폴더 ID와 최종 서명부 저장 폴더 ID가 `00_학교설정`에 입력되어 있는지 확인합니다.
- 실제 교직원 개인정보가 샘플 파일이나 저장소 문서에 포함되지 않도록 확인합니다.
