# 개인정보 보호 원칙

학교 교직원 교육센터는 개인정보와 민감한 증빙 자료를 학교별 Google Sheet와 Google Drive에만 저장하는 것을 기본 원칙으로 합니다.

## 저장 위치

- 교직원 명단: 학교별 Google Sheet
- 인증코드: 학교별 Google Sheet
- 교육대상: 학교별 Google Sheet
- 출석기록: 학교별 Google Sheet
- 전자서명 이미지: 학교별 Google Drive 폴더
- 이수증 파일: 학교별 Google Drive 폴더
- 최종 서명부: 학교별 Google Drive 폴더

## GitHub Pages 저장 금지

GitHub Pages에는 다음 정보를 저장하지 않습니다.

- 교직원 명단
- 인증코드
- 출석기록
- 전자서명 이미지
- 이수증 파일
- 최종 서명부 원본
- 학교별 민감 운영 데이터

GitHub Pages는 정적 화면만 제공합니다. 데이터 조회와 저장은 학교별 Apps Script Web App을 통해 처리합니다.

## Vercel 저장 금지

Vercel은 기본 배포 방식에서 제외합니다. 추후 고급형 UI로 사용하더라도 다음 정보를 저장하지 않습니다.

- 교직원 명단
- 인증코드
- 출석기록
- 전자서명 이미지
- 이수증 파일
- 최종 서명부 원본

## API 응답 최소화

Apps Script API는 화면 처리에 필요한 최소 정보만 반환합니다.

- 인증 성공 시에도 전체 교직원 명단을 반환하지 않습니다.
- 다른 교직원의 이수현황을 일반 화면에 노출하지 않습니다.
- 전자서명 이미지 원본 데이터를 GitHub Pages에 저장하지 않습니다.
- 오류 메시지와 로그에는 개인정보를 포함하지 않습니다.

## 문서와 샘플 데이터 원칙

- 실제 교직원 이름, 연락처, 인증코드를 저장소에 넣지 않습니다.
- 샘플 데이터는 익명 또는 가상 데이터만 사용합니다.
- Apps Script URL은 예시값만 문서화하고 실제 학교 URL은 커밋하지 않습니다.
